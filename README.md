# Proyecto Android con Git

Este proyecto fue creado como ejercicio práctico para aprender el uso de Git y GitHub junto con Android Studio.

## Objetivo

Crear un proyecto Android y aplicar control de versiones utilizando Git, trabajando con ramas, commits, merge y Pull Request.

## Herramientas utilizadas

- Android Studio
- Git
- GitHub

## Pasos realizados

1. Se creó un repositorio en GitHub llamado **mi-repositorio**.
2. Se clonó el repositorio en la máquina local.
3. Se creó un proyecto Android dentro del repositorio.
4. Se configuró un archivo **.gitignore** para evitar subir archivos innecesarios.
5. Se realizó el primer commit del proyecto base.
6. Se creó una nueva rama llamada **nueva-funcionalidad**.
7. En la nueva rama se agregó una funcionalidad al proyecto.
8. Se realizaron commits para registrar los cambios.
9. Se realizó un **merge** de la rama `nueva-funcionalidad` hacia la rama principal `main`.
10. Se subieron los cambios al repositorio remoto en GitHub.
11. Se creó un **Pull Request** para revisar los cambios.
12. Finalmente se aprobó y fusionó el Pull Request.

## Funcionalidad agregada

Se agregó una modificación en la interfaz de la aplicación Android como ejemplo de nueva funcionalidad.
